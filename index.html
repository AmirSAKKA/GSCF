Welcome to Generic Study Capture Framework

GSCF is the central module in the Phenotype Database software suite. It is a web application, written in Grails, that hosts study design information for the the studies that are stored in Phenotype Database. Also, it exposes a user interface to create and modify those study designs, to import and export data, and to query and visualize study data.

Set up a GSCF development environment

To set up your own development environment, make sure that you have a recent Java Development Kit (JDK, => 1.6) installed. Also, it depends on Grails 1.3.9, which you can download from http://grails.org and just unzip in a target directory. Make sure that JAVA_HOME and GRAILS_HOME environment variables are set to the respective locations, and that their /bin directories are in your path. Finally, it is recommended to declare a decent amount of PermGenSpace (> 150 MB) for your JVM, because both Grails and GSCF heavily use metaprogramming, and you might run out of it with the standard JVM settings.

For UNIX, those commands could for example be:

$ export JAVA_HOME="/System/Library/Frameworks/JavaVM.framework/Home"
$ export GRAILS_HOME="/app/grails-1.3.9"
$ export PATH=$PATH:$JAVA_HOME/bin:$GRAILS_HOME/bin
$ export JAVA_OPTS="-Xms256m -Xmx512m -XX:MaxPermSize=200m -XX:MaxHeapFreeRatio=70 -XX:MaxGCPauseMillis=10 -XX:+UseConcMarkSweepGC"

The shown JAVA_HOME location is the default JDK location in Mac OSX 10.6. For Windows, you can add these options by replacing 'export' with 'set' via commandline, or using the Control Panel or Computer Settings, see the Microsoft Knowledge Base (e.g. http://support.microsoft.com/kb/310519).

If you have git installed, you can checkout the source from GitHub, otherwise download the zip file or tarball at the top of this page and unzip it.

$ git clone https://github.com/PhenotypeFoundation/GSCF.git

Finally, you can run the application using Grails:

$ cd GSCF
$ grails run-app

This command should pull in all dependencies and start the project. When it is started, you can browse to http://localhost:8080/gscf to use the application. If you want to use an IDE to edit the code, you could try IntelliJ IDEA, SpringSource Tool Suite, or Netbeans.
